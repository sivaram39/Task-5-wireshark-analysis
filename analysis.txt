Task 5: Capture and Analyze Network Traffic Using Wireshark

Objective:
To capture live network traffic using Wireshark and identify common network protocols and traffic types.

Tool Used:
- Wireshark

Procedure:
1. Installed Wireshark on the system.
2. Started packet capture on the active Wi-Fi network interface.
3. Generated network traffic by browsing websites and using the ping command.
4. Captured packets for approximately one minute.
5. Stopped the packet capture.
6. Applied protocol filters to analyze the captured packets.
7. Identified different network protocols.
8. Saved the capture as a .pcap file.

Protocols Identified:

1. DNS (Domain Name System)
- Used to translate domain names into IP addresses.
- Observed when accessing websites.

2. TCP (Transmission Control Protocol)
- Provides reliable communication between devices.
- Used for establishing connections and transferring data.

3. TLS/HTTPS
- Encrypts communication between the client and web server.
- Ensures secure transmission of web traffic.

Packet Details:
- Capture Duration: Approximately 1 minute
- Network Interface: Wi-Fi
- Protocols Observed: DNS, TCP, TLS/HTTPS
- Packet Capture File: task5_capture.pcap

Conclusion:
The network traffic was successfully captured and analyzed using Wireshark. Different protocols such as DNS, TCP, and TLS/HTTPS were identified during the capture. This task provided practical experience in monitoring network traffic, understanding packet communication, and analyzing commonly used network protocols.
